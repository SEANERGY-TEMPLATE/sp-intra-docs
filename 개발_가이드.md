# 신규 기능 개발 가이드

> 2명이 기능을 나눠서 개발하는 가이드. 이 문서를 읽고 본인이 A/B 중 누구인지 확인 후 작업 시작.

---

## 1. 나는 누구인가?

### A 개발자 — 주문/UI 담당

| 기능 | 설명 | 주요 수정 파일 |
|------|------|---------------|
| **NEW-001** 자동 대문자 변환 | 저장 시 ToUpper() | OrderController, SalesOrderController |
| **NEW-005** 지급기한 태그 | 색상 뱃지 표시 | OrderList.razor, SalesOrderList.razor, 신규 컴포넌트 |
| **NEW-006** 바지선 검색 개선 | 검색 로직 수정 | OrdersController, SalesOrdersController |
| **NEW-008** 요청사항 자동 높이 | textarea 스크롤 제거 | OrderList.razor, SalesOrderList.razor |
| **NEW-009** 검색 부분 매칭 점검 | 항구명 등 %LIKE% 수정 | OrdersController, SalesOrdersController |

### B 개발자 — 인보이스/메일/파싱 담당

| 기능 | 설명 | 주요 수정 파일 |
|------|------|---------------|
| **NEW-002** AED 디르함 통화 | 인보이스 통화 선택 | InvoiceController, PdfMakerController |
| **NEW-003** IBK 은행 선택 | 인보이스 은행 선택 | InvoiceController, BankAccount 테이블 신규 |
| **NEW-004** 이메일 자동 매핑 | 이메일 자동 채움 | MailController, ManagedCompanyController |
| **NEW-007** 인쿼리 자동 파싱 | 텍스트→주문 생성 | 신규 InquiryParseController |

**이렇게 나눈 이유:** A와 B가 같은 파일을 동시에 수정하는 경우를 최소화하기 위해.

## 4. 개발 순서 (권장)

### A 개발자

```
1순위: NEW-001 (자동 대문자) — 난이도 낮음, 워밍업
2순위: NEW-008 (요청사항 자동 높이) — 프론트만, 간단
3순위: NEW-005 (지급기한 태그) — UI 컴포넌트 신규
4순위: NEW-006 (바지선 검색) + NEW-009 (검색 점검) — 같이 하면 효율적
```

### B 개발자

```
1순위: NEW-003 (IBK 은행) — DB 테이블 + 체크박스, 비교적 단순
2순위: NEW-002 (AED 디르함) — NEW-003과 비슷한 패턴
3순위: NEW-004 (이메일 매핑) — 조사 필요 (벙커 아카이브 연동)
4순위: NEW-007 (인쿼리 파싱) — 가장 큰 기능, 마지막에
```

> NEW-002와 NEW-003은 인보이스 쪽이라 같이 하면 효율적. NEW-007은 가장 크니까 마지막에.

  - 뭘 했는지

1. PR 생성

5. 머지 후 상대방에게 알림 ("main에 NEW-XXX 머지했어요, pull 받으세요")

## 9. 충돌 해결

### 같은 파일을 둘 다 수정한 경우

```bash
git merge main
# 충돌 발생 시
# 1. 충돌 파일 열기
# 2. <<<<<<< / ======= / >>>>>>> 마커 찾기
# 3. 두 변경사항 모두 반영하도록 수정
# 4. 저장 후 커밋
git add 충돌해결한파일
git commit -m "merge: main 충돌 해결"
```

### 충돌 위험이 높은 파일

| 파일 | A 수정 | B 수정 | 충돌 가능성 |
|------|--------|--------|------------|
| OrderList.razor | NEW-005 태그 | - | 낮음 |
| SalesOrderList.razor | NEW-005 태그 | - | 낮음 |
| OrderController.cs | NEW-001 대문자 | - | 낮음 |
| InvoiceController.cs | - | NEW-002, NEW-003 | **B 내부 충돌 주의** |
| EtcController.cs | - | NEW-002 환율 | 낮음 |
| Program.cs | - | NEW-003 DI 등록 | 낮음 |

> A와 B가 겹치는 파일이 거의 없으므로 충돌 가능성 낮음.
> B 개발자는 NEW-002, NEW-003 모두 인보이스 쪽이라 순서대로 하면 충돌 없음.

### 특히 주의할 점

**BuyPrice / SellPrice 헷갈리지 않기!**

```
BuyPrice  = 바이어 단가 = 선주가 사는 가격 = 우리 매출 (TotalSalesAmount)
SellPrice = 셀러 단가   = 공급사가 파는 가격 = 우리 매입 (TotalPurchaseAmount)
```

이름과 실제 의미가 반대이므로 항상 확인.
