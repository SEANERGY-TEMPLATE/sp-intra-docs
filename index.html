배경/목적

해운업에서 선박명, 항구명, 업체명 등은 관례상 대문자(UPPER CASE)로 작성. 현재는 수동 입력이라 소문자가 섞여 들어가는 경우 많음.

상세 요구사항

  1. 바이어/셀러 관리에서 저장(PUT/POST) 시 문자열 필드를 자동 대문자 변환
  2. 예외 필드: 담당자명 (이름 원형 유지), 이메일 주소
  3. 대문자 변환 대상: 선박명, 항구명, 업체명, 터미널명, 유종명, 메모/비고
  4. 백엔드 API에서 변환 (저장 시점)

영향 범위

  • 바이어 관리 — OrderController PUT/POST
  • 셀러 관리 — SalesOrderController PUT/POST
  • 정산 관리
  • 이메일 시스템
  • 업체 관리
  • 회원 관리
  • DB 스키마 변경
  • 기존 API 수정

API 설계

MethodEndpoint설명신규/수정

배경/목적

아랍권(UAE 등) 거래 시 인보이스를 디르함(AED)으로 발행해야 함. 현재 USD만 지원. 체크박스로 AED 변환 발행 가능해야 함.

상세 요구사항

  1. 인보이스/BDR 생성 화면에 "AED(디르함) 발행" 체크박스 추가
  2. 기본값: 해제 (USD 기본)
  3. 체크 시: USD → AED 환율 적용 변환 후 발행
  4. 인보이스 PDF에 통화 단위 표시: USD → AED
  5. AED 환율: 고정(3.6725) 또는 실시간 선택

영향 범위

  • 바이어 관리
  • 셀러 관리
  • 문서/파일 — 인보이스 PDF 생성 로직
  • DB 스키마 변경 — 환율 테이블 AED 추가
  • 신규 API — AED 환율 조회
  • 기존 API 수정 — 인보이스 생성 API에 currency 파라미터

API 설계

MethodEndpoint설명신규/수정
POST/api/invoice/Generatecurrency 파라미터 추가 (USD/AED)수정
POST/api/invoice/PreviewOrdercurrency 파라미터 추가수정
GET/api/etc/ExchangeRateAED 환율 지원 추가수정

참고

USD/AED 고정환율: 1 USD = 3.6725 AED (UAE 페그제). 큰 변동 없음.

배경/목적

인쿼리 텍스트를 보고 수동으로 바이어/셀러 주문을 등록 중. 자동 파싱하여 주문을 자동 생성해야 함.

입력 예시

VESSEL NAME: TONG RUN 7 IMO : 9673563 DELIVERY PORT/AREA NAME: BUSAN DELIVERY WINDOW: 16-20TH MAR 2026 VLSFO : 55~100MT LSMGO : 22~200MT 바이어 : HONGKONG E-MARINE ... 1370/1620+500 (30DDD) 셀러 : KEUMJIN MARINE ... 1290/1540+500 (7DDD) AGENT : GLOBAL OFFSHORE MARINE SERVICE

파싱 매핑

텍스트 키매핑 대상비고
VESSEL NAMEShip.ShipNameDB 부분 매칭
IMOShip.Imo정확 일치
DELIVERY PORTOrderInfo.PortNameDB 부분 매칭
DELIVERY WINDOWReqSupplyStartDate ~ EndDate날짜 파싱
VLSFO, LSMGO 등OrderOil (유종+수량)55~100MT → Min/Max
바이어ShipCompany부분 매칭 + 단가/결제조건
셀러OilSalesCompany부분 매칭 + 단가/결제조건
AGENTAgent부분 매칭

단가/결제조건 파싱

1370/1620+500 (30DDD) ├─ 1370 → 첫 번째 유종(VLSFO) BuyPrice ├─ 1620 → 두 번째 유종(LSMGO) BuyPrice ├─ +500 → AddPay (부대비용) └─ 30DDD → PayType (30일 결제)

처리 순서

  1. 텍스트 입력
  2. 파싱 → 구조화된 데이터로 변환
  3. DB 매칭 (업체, 선박, 항구, 유종 부분 매칭)
  4. 미리보기 표시 (매칭 결과 + 미매칭 하이라이트)
  5. 사용자 확인/수정
  6. 바이어 주문 생성 (POST /api/order/OrderForAdmin)
  7. 셀러 주문 생성 (POST /api/Sales/Order/AddExclusiveReg) — 바이어 OrderInfoId 연결

영향 범위

  • 바이어 관리 — 주문 자동 등록
  • 셀러 관리 — 판매주문 자동 등록
  • 업체 관리 — 업체 매칭 조회
  • 회원 관리 — 담당자 매칭 조회
  • 견적(RFQ) — 인쿼리 입력 화면
  • 신규 API
  • 기존 API 수정

API 설계
